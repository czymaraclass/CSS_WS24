2. Structural Topic Modelling

a) Run Model

Now that we have prepared the data, we can run a topic model. Use the stm() function to run a model with 20 topics that includes genre as a document-level covariate for topic prevalence. This allows to examine how topic prevalence varies across genres. Run the STM model and use summary() to examine the topics. What themes emerge in the identified topics?

k <- 20 # number of topics

set.seed(1337)
news_topics <- stm(newspapers_dfm,
                     K = k,
                     prevalence = ~ genre,
                     verbose = FALSE

First, we need to run a regression that estimates the relationships between topic probabilities and the genre covariate. Use stmâs estimateEffect() function to run such a regression, based on the topic model of 2.a).

genre_diff <- estimateEffect(1:k ~ genre,

4. Choosing the Number of Topics

Finally, we want to analyze whether running a model with 20 topics was actually a good decision. Note that, usually, you would explore the number of topics before running the model.

a) Searching for the Optimal Topic Number

Run searchK() to determine the optimal number of topics for this data. This involves examining various metrics, such as:

  • Exclusivity: How distinct are the words for each topic?
    • Maximize (make topics more unique and clearly separated)
  • Semantic Coherence: How often the most important words for a topic appear together in the data
    • Maximize (make topics more meaningful and interpretable)
  • Heldout Likelihood: Tests how well the model predicts new, unseen data
    • Maximize (improve how well the model generalizes to new data)
  • Bound: Measures how well the model fits the data
    • Typically maximized during training
  • Lower bound: - A refined version of the Bound, which adjusts for comparison across models
    • Maximize (better model fit)
  • Residual Dispersion: - Checks how much of the dataâs variability remains unexplained by the topics
    • Minimize (reduce unexplained variability)

First, run out <- convert(newspapers_dfm, to = "stm"). Then, use searchK() on this out object. Experiment with different topic numbers (e. g., 10, 20, 30).

out <- convert(newspapers_dfm, to = "stm")

set.seed(1337)
kresult <- searchK(out$documents,
                   out$vocab,
                   seq(10, 30, 10), # 10 to 30, in 10
                   prevalence = ~ genre,
                   data = out$meta,
                   verbose = FALSE
                   )

b) Visualizing Search Results

plot() the results of 4.a) to evaluate the metrics. Interpret this plot.

plot(kresult)

c) Evaluating Exclusivity and Semantic Coherence

Another way to decide on the number of topics is plotting exclusivity against semantic coherence. Exclusivity measures how unique a topicâs words are (high values meaning that the words are less likely to overlap with other topics) and semantic coherence measures how often a topicâs words appear together in the same text (high values implying that words are contextually related). By plotting these two metrics, we can balance the trade-off between distinctness (exclusivity) and interpretability (semantic coherence). A good choice for the number of topics typically optimizes both.

Run the following code and interpret the result. Which number of topics would you chose?

plot(kresult$results$exclus, kresult$results$semcoh,
     xlab = "Semantic Coherence",
     ylab = "Exclusivity")


text(kresult$results$exclus, kresult$results$semcoh, 
     labels = kresult$results$K, pos = 4)
